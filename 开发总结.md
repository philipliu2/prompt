## 遇到的问题与解决方案

### 问题1：SyntaxError - 嵌入 JS 导致语法错误

**现象：**
- `Uncaught SyntaxError: Invalid or unexpected token at line 985`
- 提示词 text 字段包含特殊字符导致 JS 解析失败

**原因：**
- 尝试用 Python 脚本将 prompts 嵌入到 HTML 的 JS 中
- Python 的字符串转义与 JavaScript 不兼容
- 文本中的换行符、引号等未被正确处理

**解决方案：**
- 改用 `prompts.json` 纯 JSON 文件存储
- 通过 `fetch()` 动态加载
- 不再嵌入提示词到 HTML

```javascript
// 加载逻辑
if (stored) {
    prompts = JSON.parse(stored);
} else {
    const res = await fetch('prompts.json?t=' + Date.now());
    prompts = await res.json();
    savePrompts();
}
```

### 问题2：浏览器缓存导致加载旧数据

**现象：**
- 更新 prompts.json 后，用户看到的是旧数据

**解决方案：**
- 添加时间戳 cache-busting
```javascript
const res = await fetch('prompts.json?t=' + Date.now());
```

### 问题3：generate_index.py 脚本污染 template.html

**现象：**
- 运行脚本后，Python 代码残留在 HTML 中
- template.html 被破坏

**原因：**
- 脚本查找替换 SAMPLE_PROMPTS 数组时出错
- 残留了 Python 代码在 HTML 中

**解决方案：**
- 从 git 历史恢复 clean 版本
- 重新应用 SPEC 修改
- 不再使用 generate_index.py

### 问题4：fetch prompts.json 不工作

**现象：**
- 网站加载后没有显示任何提示词

**排查过程：**
1. 确认 prompts.json 文件存在且有效
2. 确认 GitHub Pages 上可访问
3. 添加 console.log 调试

**解决方案：**
```javascript
async function loadPrompts() {
    // 添加 try-catch 和详细错误日志
    try {
        const res = await fetch('prompts.json?t=' + Date.now());
        if (!res.ok) throw new Error('HTTP ' + res.status);
        SAMPLE_PROMPTS = await res.json();
        prompts = SAMPLE_PROMPTS.map(p => ({...p, likedByMe: false}));
        savePrompts();
        console.log('Loaded', prompts.length, 'prompts from prompts.json');
    } catch (e) {
        console.error('Failed to load prompts:', e);
        prompts = [];
    }
}
```

### 问题5：npm 项目名称限制大写

**现象：**
- `npx create-next-app AndyAI` 报错："name can no longer contain capital letters"

**解决方案：**
- 使用小写名称创建临时目录
- 创建完成后重命名为大写

## 经验教训

### 1. 数据存储选择

**教训：**
- 静态网站不应将大量数据嵌入 JS
- 嵌入会导致语法错误和维护困难

**正确做法：**
- JSON 文件 + fetch 动态加载
- 或使用 localStorage（适合少量用户私有数据）
- GitHub 无法直接编辑 JSON，需要本地修改后 push

### 2. 模板文件保护

**教训：**
- template.html 是重要的源文件
- 运行脚本前应备份
- 脚本应该只修改 index.html，不应修改 template.html

**正确做法：**
- template.html 保持 clean
- 脚本只操作 index.html
- 或完全不用脚本，直接 fetch JSON

### 3. Git 提交粒度

**教训：**
- 一次性提交大量临时文件

**正确做法：**
- 及时清理临时文件
- 保持 commit 整洁
- 每个功能单独的 commit

### 4. 浏览器缓存

**教训：**
- 静态资源会被缓存
- 用户可能看不到更新

**正确做法：**
- 添加版本号或时间戳
- 或使用 Vercel/Netlify 等会自动刷新的平台

### 5. GitHub Pages vs Vercel

**比较：**
| 特性 | GitHub Pages | Vercel |
|------|-------------|---------|
| 价格 | 免费 | 免费（有额度）|
| 自动部署 | 是 | 是 |
| 自定义域名 | 支持 | 支持 |
| HTTPS | 自动 | 自动 |
| SSR 支持 | 否 | 是 |
| 预览部署 | 否 | 是 |

**PromptWorld 选择 GitHub Pages 的原因：**
- 纯静态网站，不需要 SSR
- 项目已在 GitHub
- 简单够用

## 总结

PromptWorld 是一个成功的静态网站项目，核心功能完整。但由于提示词数据存储方式的限制（JSON 文件无法直接在 GitHub 网页编辑），数据更新需要本地修改后 push，这在一定程度上影响了协作效率。

未来的改进方向可以是：
1. 使用 Vercel + JSON 文件，实现更便捷的部署
2. 添加 CMS 功能，允许在网页上直接编辑
3. 添加用户系统，实现真正的多人协作
